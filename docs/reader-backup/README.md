## Purpose

This folder documents the working EPUB reader implementation after the column-based pagination fix. It is intended as a recovery and comparison reference before later reader UI, panel, menu, settings, table-of-contents, or bookmark refactors.

This is not runtime application code. It is a documentation backup only.

## Fixed In This Baseline

- Long inline paragraphs can continue across reader pages.
- Genesis 1 in the default Bible no longer clips content.
- Desktop two-page spread mode no longer leaves the left page as only a chapter title while clipping the right page.
- Mobile single-page mode continues within the same chapter instead of jumping ahead.
- The bottom reader bar is accounted for by the page metrics and should not cover readable text.
- Long-chapter EPUB files can be paged continuously.

## Files

- `reader-appjs-snapshot.md`: reader state, open/read flow, pagination functions, and EPUB loading/parsing notes from `app.js`.
- `reader-styles-snapshot.md`: reader CSS selectors and layout rules, including runtime injected CSS from `injectReaderViewStyles()`.

## Recovery Notes

If a later reader refactor regresses pagination, use Git to return to this commit or compare the snapshots in this folder against the changed implementation.

Important invariants to preserve:

- Pagination must be based on the rendered page body width and height.
- `.reader-column-flow` must remain column-based and offset by page index.
- `.reader-column-page`, `.reader-page-clip`, and `.reader-page-surface` intentionally use clipping; continuation pages must be generated by pagination, not by scrollbars.
- Desktop spreads show two page indexes at a time; mobile shows one.
- Resizing, font-size changes, and line-height changes must rebuild pagination and preserve approximate progress.
